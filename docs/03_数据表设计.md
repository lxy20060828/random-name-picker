# 随机点名器 数据表设计

## 1. 数据存储方案

- 存储方式：localStorage
- 存储键：
  - `random-picker-vue3-students`：学生名单
  - `random-picker-vue3-history`：点名记录
- 数据结构概述：项目不连接后端数据库，所有数据以 JSON 字符串形式保存在浏览器 localStorage 中。刷新页面后会自动读取本地数据。

## 2. 实体关系图（文字）

```text
Student 1 ──── N PickHistory

一个学生可以对应多条点名记录；
一条点名记录只对应一个被点名学生；
一个学生可以同时属于多个班级、课程和标签。
```

## 3. 数据表结构

### 表名：Student

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|:---|:---|:---:|:---:|:---|
| id | string | 是 | uuid | 学生唯一标识 |
| name | string | 是 | 空 | 学生姓名 |
| studentNo | string | 是 | 空 | 学号，作为主要唯一标识 |
| grade | string | 否 | 空 | 年级，例如 2026级 |
| department | string | 否 | 空 | 院系或学院 |
| major | string | 否 | 空 | 专业 |
| classes | string[] | 是 | [] | 班级列表，可包含行政班、教学班 |
| courses | string[] | 是 | [] | 课程列表 |
| tags | string[] | 是 | [] | 标签或分组 |
| note | string | 否 | 空 | 备注 |
| pickCount | number | 是 | 0 | 被点名次数 |
| createdAt | string | 是 | 当前时间 | 创建时间 |
| updatedAt | string | 是 | 当前时间 | 更新时间 |

合并规则：

- 导入数据有学号时，优先按 `studentNo` 查找已有学生。
- 同一学号且姓名一致时，补充新增的 `classes`、`courses`、`tags`，不会重复创建。
- 同一学号但姓名不一致时，保留原学生，并在导入结果中提示冲突。
- 无学号时按姓名兜底匹配；如果本地已有多个同名学生，则跳过并提示人工处理。
- 旧版本数据中的 `className` 会在读取时迁移为 `classes: [className]`。

### 表名：PickHistory

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|:---|:---|:---:|:---:|:---|
| id | string | 是 | uuid | 记录唯一标识 |
| studentId | string | 是 | 空 | 被点名学生 id |
| studentName | string | 是 | 空 | 被点名学生姓名 |
| pickedAt | string | 是 | 当前时间 | 点名时间 |

### 表名：Quote

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|:---|:---|:---:|:---:|:---|
| content | string | 是 | 本地备用语句 | 每日一句内容 |
| source | string | 是 | 本地备用语句 | 来源 |
| fromApi | boolean | 是 | false | 是否来自外部 API |
